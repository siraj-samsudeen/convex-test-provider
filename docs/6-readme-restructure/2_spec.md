# README Restructure Specification

> GitHub Issue: README restructure for clarity

## Requirements

### R1: Table-first hero
The README must open with a comparison table showing the 4 testing approaches and what each covers.

### R2: Narrative grounding
Immediately after the table, a brief narrative explains each approach using a concrete `<TodoList>` example, naming what each tests and what it misses.

### R3: Name the gap, show the solution
A clear "gap" statement followed by a 4-line code example showing how this library fills it.

### R4: Preserve existing content
All existing features, API reference, and configuration docs must be preserved — just reorganized below the hero.

## Acceptance Criteria

- [ ] First thing a reader sees is the comparison table
- [ ] The gap is named within the first screenful
- [ ] Code example appears within the first screenful
- [ ] All existing content preserved (features, API ref, config, etc.)
